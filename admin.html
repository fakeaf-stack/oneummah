Upload Products

Add a product once and it will show up on the homepage, catalog, and product detail page automatically after the page reloads.

Use this page only for trusted uploads. The browser can use the public anon key, but your Supabase table and storage bucket must allow the insert/upload rules you want. If the service role key was exposed earlier, rotate it in Supabase before going live.

New Product

Select one or more product images to preview them here before upload.
Likely cause: Row-Level Security (RLS) is enabled for the products table and prevents anonymous inserts from the browser.

If you want to allow this quick admin upload from your browser (temporary), run the SQL below in the Supabase SQL editor, then retry the upload. Remove or tighten this policy when finished.

-- Allow INSERT into products and product_images for the anon role (temporary)
CREATE POLICY allow_insert_products_anon ON public.products
  FOR INSERT
  TO anon
  USING (true)
  WITH CHECK (true);

CREATE POLICY allow_insert_product_images_anon ON public.product_images
  FOR INSERT
  TO anon
  USING (true)
  WITH CHECK (true);

-- NOTE: For secure production use, do not leave these policies wide-open.
-- Prefer creating a server endpoint that uses a service_role key or
-- require authentication and create policies that check auth.uid().

What gets saved

The product goes into the products table. Every selected image uploads into the product-images storage bucket, then each public URL is stored in product_images.

Automatic updates

After upload, refresh the homepage or catalog. Both pages already load products from Supabase and will show the new item without manual HTML edits.

Detail page

Clicking a product opens product.html?id=..., which shows a larger image, description, sizes, and gallery thumbnails.
